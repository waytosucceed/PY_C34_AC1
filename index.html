EXCEPTION

An exception is an error that happens during execution. It is Python’s way of saying:

“Hey! Something unexpected just happened, and I don’t know how to continue!”

It could be:
  • A bad input
  • Dividing by zero
  • Accessing a file that doesn't exist
  • Using the wrong data type
Lots of things can go wrong.

WHY IS EXCEPTION HANDLING IMPORTANT?

So now that we know what an exception is, the next big question is:
What should we do when that happens? Should we just let the program crash?

Of course not!

That’s exactly why we need exception handling — a way to deal with these unexpected situations without breaking our entire program.

‘Try’ AND ‘Except’
Python gives us two handy keywords for handling exceptions:
  • try
  • except
These keywords are used to define blocks of code:
  • The try block contains the code that might cause an error, and
  • The except block defines how to respond if an error occurs.

The except block is the safety net, where you decide what you want to do if something fails.

‘else’ and ‘finally’ IN EXCEPTION HANDLING
Didn’t we already use else before... with if?"
Absolutely. You're not wrong.
In Python, the word else always means the "otherwise" part — the backup plan.
But the way it’s used depends on where you put it.
In exception handling, we use else to run code only if no error happened inside the try block.

So here's the idea:
  • try - "This might go wrong"
  • except - "Oops, it went wrong — let's fix it"
  • else - "Everything went fine — go ahead with the next safe step"

You might wonder, “Why not just put print("Thank you for using the calculator.") after all the try-except blocks? Wouldn't that do the same thing?”
Most of the time, it would work. But here’s why finally is special and important:
  • Code inside the finally block always runs, no matter what — whether an exception happens or not, even if the program is interrupted early by a return, break, or another exception that isn’t caught.
  • If you put that print outside, and an unexpected error causes the program to stop immediately, that message never get printed.
  • Think of finally as a guarantee: it’s your cleanup crew, always on duty, making sure important tasks get done your program exits a risky block of code.
